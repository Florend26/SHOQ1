## Design Direction

- Product type: centralized student academic portal.
- Visual feel: clean, modern, university-facing, dashboard-first.
- Primary users: students.
- Secondary users: professors, IT support, administrative staff.
- Core problem: academic services are split across different platforms, making grades, exams, schedules, materials, and notices harder to find.
- Core promise: one unified dashboard where students can access academic information quickly and clearly.

## Must-Have Features

| Feature | Why it matters | Priority |
|---|---|---|
| Centralized dashboard | Solves scattered information across platforms | High |
| Learning materials access | Keeps course literature and materials in the main system | High |
| Accessibility assistant | Helps more students use the system clearly and comfortably | Medium |
| Modern clean interface | Makes the portal easier and more attractive to use | Medium |

## Target Users

| Segment | Need | Pain Point |
|---|---|---|
| Students | Fast access to grades, exams, registrations, materials, and academic info | Information is scattered across platforms |
| IT / Technical Support | Easier technical maintenance and support | Separate systems create complexity |
| Professors | Easy publishing of grades, notices, course materials, and academic updates | Lack of one central publishing system |

## Dashboard Screen Concept

Create a desktop frame at 1440x1024 named `Unified Student Dashboard`.

Layout:
- Left sidebar: UBT / Student Portal logo, Dashboard, Grades, Exams, Schedule, Materials, Notices, Accessibility.
- Top bar: page title, search field, notification button, student profile.
- Main grid:
  - Welcome panel with student name and semester summary.
  - Quick cards: Grades, Exams, Schedule, Materials.
  - Upcoming deadlines and exam registration card.
  - Recent course materials list.
  - Notifications panel.
  - Accessibility assistant floating action button or right-side panel.

Suggested color system:
- Background: off-white / very light gray.
- Primary: university blue.
- Accent: green for success/completed states.
- Warning: amber for deadlines.
- Text: dark neutral.
- Borders: soft neutral gray.

Typography:
- Headings: Inter or similar sans serif, semibold.
- Body: Inter regular.
- Use clear hierarchy for scanning.
